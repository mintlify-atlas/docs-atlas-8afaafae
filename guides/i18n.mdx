---
title: "Internationalization with ConfigProvider"
description: "Configure antdv-next for any of 70+ supported languages using the locale prop on ConfigProvider. Switch locales at runtime with Vue reactivity."
---

antdv-next ships with English as the default language, but it includes built-in locale objects for more than 70 languages. Every piece of UI text produced by the component library — button labels in date pickers, empty-state messages, validation text, pagination labels — is controlled through a single `locale` prop on `ConfigProvider`. Because `locale` is reactive, you can switch languages at runtime with no page reload.

## Using ConfigProvider with a locale

Wrap your application root with `a-config-provider` and pass a locale object imported from the `antdv-next/locale/` directory. The filename follows the pattern `language_REGION` (for example `fr_FR` for French or `zh_CN` for Simplified Chinese).

```vue
<script lang="ts" setup>
import frFR from 'antdv-next/locale/fr_FR'
</script>

<template>
  <a-config-provider :locale="frFR">
    <App />
  </a-config-provider>
</template>
```

## Switching locale at runtime

Because `locale` is a reactive prop, you can drive it from a `ref` or `computed` and change it in response to user input. Vue's reactivity system propagates the new locale to every component in the tree automatically.

```vue
<script lang="ts" setup>
import { ref } from 'vue'
import enUS from 'antdv-next/locale/en_US'
import zhCN from 'antdv-next/locale/zh_CN'
import frFR from 'antdv-next/locale/fr_FR'

const locales = { enUS, zhCN, frFR } as const
type LocaleKey = keyof typeof locales

const currentLocale = ref<LocaleKey>('enUS')
</script>

<template>
  <div>
    <a-space>
      <a-button @click="currentLocale = 'enUS'">English</a-button>
      <a-button @click="currentLocale = 'zhCN'">中文</a-button>
      <a-button @click="currentLocale = 'frFR'">Français</a-button>
    </a-space>

    <a-config-provider :locale="locales[currentLocale]">
      <App />
    </a-config-provider>
  </div>
</template>
```

## Date picker and dayjs

`ConfigProvider`'s `locale` prop controls the UI strings rendered by antdv-next components. The date picker also relies on [dayjs](https://day.js.org) to format and parse dates, and dayjs has its own separate locale system.

<Note>
  To fully localize date and time formatting in `DatePicker` and `TimePicker`, you must also import and apply the matching dayjs locale. The `ConfigProvider` locale alone does not change how dayjs formats dates.

  ```ts
  import dayjs from 'dayjs'
  import 'dayjs/locale/fr'

  dayjs.locale('fr')
  ```
</Note>
